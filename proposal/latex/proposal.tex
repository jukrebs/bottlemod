\section{Motivation}

Data-intensive workloads are increasingly bottlenecked by storage hierarchy effects that current performance models ignore. Scientific workflows in genomics, satellite image processing, and video analytics spend significant time on I/O operations where execution time depends on whether data resides in the page cache or must be fetched from block devices. Current models typically treat storage as a single throughput number, ignoring the tiered nature of modern storage systems. This leads to incorrect bottleneck attribution and inaccurate runtime predictions for cache-sensitive workloads. Accurate modeling of storage hierarchy effects would enable schedulers to make informed decisions about resource allocation and improve workflow execution times.

\section{Related Work}

BottleMod~\cite{bottlemod} introduces a progress-based bottleneck model using requirement and input functions to predict task execution analytically. SimGrid~\cite{simgrid} is a versatile simulation framework for distributed applications using discrete-event simulation. WRENCH~\cite{wrench}, built on SimGrid, models workflows as read-compute-write processes but lacks temporal task behavior detail. Stack distance analysis~\cite{mattson} provides a classic cache modeling technique using reuse distance to predict hit rates. The Linux page cache is the kernel mechanism caching file data in memory to accelerate I/O operations.

\section{Goal}

This thesis extends BottleMod to model storage hierarchy effects, specifically the interaction between page cache and block device tiers, and validates the extension through experiments demonstrating improved prediction accuracy for I/O-intensive workloads. The research problem is: given an I/O-intensive task, how can we accurately predict its runtime when execution time depends on cache hit rates and tier bandwidths? The baseline BottleMod model cannot represent warm-cache speedups or tier shifts caused by working set sizes exceeding cache capacity. The scope includes single-node storage behavior, Linux page cache modeling, and bandwidth constraints; distributed storage systems and network effects are out of scope.

\section{Implementation and Experiment}

The BottleMod-CA extension adds three constructs to the baseline model. The LogicalAccessProfile defines bytes read and written over progress for each dataset. The TierMapping provides cache hit rate functions determining what fraction of accesses are served by each storage tier. The StorageTier defines bandwidth constraints for each tier in the storage hierarchy. The progress solver remains unchanged; only the resource requirement functions are expanded to include tier-specific constraints.

We validate the cache-aware extension using an ffmpeg video remux workload (4.6~GB video on SATA disk), with cgroup memory limits from 256~MB to 16~GB to vary the cacheable working set. Calibration yields disk bandwidth of 242.7~MB/s and memory bandwidth of 350.7~MB/s. The vanilla BottleMod model predicts a constant runtime of approximately 18.9~s regardless of cache state. The cache-aware model predicts runtimes from 18.9~s (cold cache) down to 13.1~s (warm cache, fully cached), demonstrating a 1.44$\times$ improvement with cache awareness. These results confirm that the extension correctly captures tier transitions and predicts speedups when data fits in memory, while the vanilla model remains oblivious to cache effects.
